# Локализация (i18n)

Проект поддерживает два языка: русский и английский. Язык по умолчанию — русский. Переключение языка осуществляется через выпадающий список в навигационной панели (иконка глобуса).

## Настройка

В [`settings.py`](../task_manager/settings.py:129):

```python
LANGUAGE_CODE = "ru"
LANGUAGES = ("ru", _("Russian")), ("en", _("English"))
LOCALE_PATHS = [BASE_DIR / "task_manager" / "locale"]
```

- `LANGUAGE_CODE = "ru"` — язык по умолчанию
- `LANGUAGES` — список доступных языков
- `LOCALE_PATHS` — путь к каталогу с файлами перевода

Middleware [`LocaleMiddleware`](../task_manager/settings.py:58) включён в `MIDDLEWARE` и автоматически определяет язык пользователя на основе сессии или cookie. Он должен располагаться после `SessionMiddleware` и перед `CommonMiddleware`.

## Файлы перевода

Переводы хранятся в формате GNU gettext (`.po` / `.mo`):

```
task_manager/locale/
├── en/
│   └── LC_MESSAGES/
│       └── django.po    # Английский перевод (250 строк)
└── ru/
    └── LC_MESSAGES/
        └── django.po    # Русский перевод (251 строка)
```

### Соглашения по именованию msgid

В проекте в качестве `msgid` используются русские фразы (не абстрактные идентификаторы). Это упрощает чтение исходного кода и позволяет использовать русский язык как базовый:

```po
msgid "Задача"
msgstr "Task"
```

Для русского перевода `msgid` и `msgstr` совпадают, так как русский — язык по умолчанию.

## Обновление переводов

```bash
# Создать/обновить .po файлы (извлечь строки из кода и шаблонов)
uv run django-admin makemessages -l ru
uv run django-admin makemessages -l en

# Скомпилировать .po в .mo (бинарные файлы для использования Django)
uv run django-admin compilemessages
```

## Переключение языка

Язык переключается через выпадающий список в правой части навигационной панели (иконка глобуса). При выборе языка отправляется `POST`-запрос на маршрут `set_language`:

```html
<form id="language-form" action="{% url 'set_language' %}" method="post">
    {% csrf_token %}
    <input type="hidden" id="language-input" name="language" value="">
</form>
```

Маршрут `/i18n/` подключён в корневом [`urls.py`](../task_manager/urls.py:22):

```python
path("i18n/", include("django.conf.urls.i18n")),
```

Django автоматически обрабатывает этот маршрут через встроенное представление `set_language`, которое сохраняет выбранный язык в сессии пользователя.

## Использование в коде

### Python

В Python-коде используется `gettext_lazy` (импортируется как `_`) для отложенного перевода — строка переводится в момент рендеринга, а не в момент определения:

```python
from django.utils.translation import gettext_lazy as _

# В моделях
verbose_name = _("Задача")

# В представлениях (flash-сообщения)
success_message = _("Вы залогинены")
_no_permissions_message = _("У вас нет прав для просмотра задач")
_test_failure_message = _("Невозможно удалить статус: существуют связанные задачи")
```

### Шаблоны

В шаблонах используются теги `{% trans %}` и `{% blocktrans %}` после загрузки пакета `i18n`:

```django
{% load i18n %}

<!-- Простой перевод -->
<h1>{% trans "Задачи" %}</h1>

<!-- Перевод с сохранением в переменную для использования в bootstrap_button -->
{% trans "Войти" as login_button_text %}
{% bootstrap_button login_button_text button_type="submit" button_class="btn-primary" %}

<!-- Перевод с интерполяцией переменной (blocktrans) -->
<p>{% blocktrans %}Вы уверены, что хотите удалить задачу{% endblocktrans %} {{ task.name }}?</p>
```

## Категории переводимых строк

Все строки в `.po`-файлах сгруппированы по тематическим категориям:

| Категория                  | Описание                      | Пример                                             |
| -------------------------- | ----------------------------- | -------------------------------------------------- |
| **Models**                 | Названия моделей и их полей   | `"Задача"`, `"Статус"`, `"Имя"`, `"Дата создания"` |
| **Header and navigation**  | Элементы навигационной панели | `"Менеджер задач"`, `"Вход"`, `"Выход"`            |
| **Index page**             | Главная страница              | `"Привет от Хекслета!"`, `"Узнать больше"`         |
| **Login**                  | Страница входа                | `"Войти"`                                          |
| **Users**                  | CRUD пользователей            | `"Регистрация"`, `"Изменение пользователя"`        |
| **Statuses**               | CRUD статусов                 | `"Создать статус"`, `"Удаление статуса"`           |
| **Labels**                 | CRUD меток                    | `"Создать метку"`, `"Изменение метки"`             |
| **Tasks**                  | CRUD задач и фильтрация       | `"Создать задачу"`, `"Только свои задачи"`         |
| **Success messages**       | Flash-сообщения об успехе     | `"Задача успешно создана"`, `"Вы залогинены"`      |
| **No permission messages** | Сообщения об отсутствии прав  | `"У вас нет прав для просмотра задач"`             |
