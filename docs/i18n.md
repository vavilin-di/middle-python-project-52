# Локализация (i18n)

Проект поддерживает два языка: русский и английский. Переключение языка осуществляется через URL-параметр.

## Настройка

В [`settings.py`](../task_manager/settings.py:128):

```python
LANGUAGE_CODE = "ru"
LANGUAGES = ("ru", _("Russian")), ("en", _("English"))
LOCALE_PATHS = [BASE_DIR / "task_manager" / "locale"]
```

- `LANGUAGE_CODE = "ru"` — язык по умолчанию
- `LANGUAGES` — список доступных языков
- `LOCALE_PATHS` — путь к каталогу с файлами перевода

Middleware (`LocaleMiddleware`) включён в `MIDDLEWARE` и автоматически определяет язык пользователя на основе сессии или cookie.

## Файлы перевода

Переводы хранятся в формате GNU gettext (`.po` / `.mo`):

```
task_manager/locale/
├── en_US/
│   └── LC_MESSAGES/
│       └── django.po    # Английский перевод (364 строки)
└── ru_RU/
    └── LC_MESSAGES/
        └── django.po    # Русский перевод (365 строк)
```

### Соглашения по именованию msgid

В проекте используются осмысленные строковые идентификаторы (не сами фразы), что упрощает поддержку при изменении формулировок:

```po
# Вместо:
# msgid "Task"
# msgstr "Задача"

# Используется:
msgid "TaskVerboseName"
msgstr "Задача"
```

## Обновление переводов

```bash
# Создать/обновить .po файлы (извлечь строки из кода и шаблонов)
uv run django-admin makemessages -l ru
uv run django-admin makemessages -l en

# Скомпилировать .po в .mo (бинарные файлы для использования Django)
uv run django-admin compilemessages
```

## Переключение языка

Язык переключается через URL-параметр (маршрут `/i18n/` в корневом `urls.py`):

```
http://localhost:8000/i18n/setlang/?language=en
http://localhost:8000/i18n/setlang/?language=ru
```

## Использование в коде

### Python

```python
from django.utils.translation import gettext_lazy as _

# В моделях
verbose_name = _("TaskVerboseName")

# Во вьюхах (flash-сообщения)
success_message = _("LoginSuccess")
```
